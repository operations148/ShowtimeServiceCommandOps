## Explicitly out of scope (confirmed exclusions from the phase-prompt's non-negotiable rule #6)

The following Markate-documented capabilities are **not** being built at any phase, matching `ADR-0001`:

- AI voice receptionist / Conversation AI (Markate's "Kate AI" and lead-capture AI receptionist)
- Autonomous customer messaging / marketing automation (Markate's email/SMS drip campaigns, ringless voicemail, postcard blasts)
- Shopify, Slack integrations
- An integration marketplace (Markate's Zapier/NiceJob/ResponsiBid-style paid-connector ecosystem)
- Native mobile applications (iOS/Android apps — ServiceOps remains a PWA/mobile-web app by design)
- Enterprise warehouse/stock-control inventory management (Markate itself only has an Items/pricebook catalog, not true inventory control, per `markate_report.md`'s own finding — so this exclusion doesn't even give up ground relative to Markate's real capability)

## Corrections to prior documentation, confirmed by independent module-by-module verification

- **GHL `ContactCreate`/`ContactUpdate`/`AppointmentBooked` webhook handlers are wired and functional**, contradicting `MEMORY.md`'s Phase 16/17 status ("Not Started" / "handler exists, not wired"). `src/app/api/ghl/webhooks/route.ts`'s dispatch switch calls `upsertPropertyFromGHL()` and `createWorkOrderFromAppointment()` for real, with tenant resolution and idempotency guards, alongside the already-known-working `OpportunityStatusChange` handler. `MEMORY.md` should be updated to reflect this (a positive correction — something documented as not-done turned out to be done).
- **The invoice/Stripe backend built in a prior session has zero callers anywhere in the app.** `createInvoiceFromEstimate` and `createDepositCheckoutSession` each appear only in their own definition file when grepped repo-wide — nothing ever invokes either function. Combined with the missing `/dashboard/invoices` page, the missing `/estimate/[token]` page, and a sidebar "Invoices" link that 404s (`src/config/navigation.ts` → `/dashboard/invoices`, which doesn't exist), the entire Phase-15 invoice feature is unreachable from every direction — not "partially wired," genuinely orphaned code today.
- **Two DB tables are entirely dead**: `checklist_items` and `technician_notes` have real migrations and were designed as the normalized/structured source of truth (per their own migration comments), but zero application code references either table — the app exclusively uses the `visits.checklist` JSONB column and the `visits.technician_notes` plain-text column instead.
- **`/dashboard/reports/va` is mislabeled** — the tab UI calls it "Tech Performance" and renders tech data there, while the real, separate VA-performance report (`/api/reports/va-performance`) is fully implemented but never called by any page.
- Two additional empty/dead route directories exist: `src/app/tech/complete/[id]` (no `page.tsx`) and `src/components/visits/` (zero files) — both consistent with features that were scaffolded but never built.
